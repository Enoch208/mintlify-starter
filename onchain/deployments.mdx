## Configuration

The app resolves on-chain identifiers from public env with a fallback to the committed deployment:

```ts
// apps/web/lib/cairn-config.ts
export const CAIRN = {
  network: process.env.NEXT_PUBLIC_CAIRN_NETWORK ?? "testnet",
  packageId: process.env.NEXT_PUBLIC_CAIRN_PACKAGE_ID ?? deployment.packageId,
  module: "access",
  policyId: process.env.NEXT_PUBLIC_CAIRN_POLICY_ID ?? deployment.accessPolicy,
  clockId: "0x6", // shared Clock, fixed on every Sui network
};
```

## Publish your own

<Steps>
  <Step title="Switch to testnet and fund gas">
    ```bash
    sui client switch --env testnet
    sui client active-address
    sui client gas   # need ~0.2 SUI; faucet at https://faucet.sui.io
    ```
  </Step>
  <Step title="Publish">
    ```bash
    cd contracts
    sui client publish --gas-budget 200000000 --json > publish.json
    ```
    Grab the new `packageId` from the published `objectChanges`.
  </Step>
  <Step title="Create your policy">
    `publish` doesn't call `create()`, so make your shared `AccessPolicy` + `OwnerCap`:
    ```bash
    sui client call --package <packageId> --module access --function create --gas-budget 50000000
    ```
  </Step>
  <Step title="Update deployments/testnet.json">
    Set `packageId`, `accessPolicy`, `ownerCap`, and `owner` to the new values — the app picks them up automatically.
  </Step>
</Steps>

<Warning>
The Cairn contract changes (new `Receipt` object, new `anchor_receipt` signature) are **breaking**, so a fresh `publish` — not an in-place `upgrade` — is required.
</Warning>
