This guide takes you from zero to a running Cairn: teach a fact, recall it across models, watch the gate block a revoked namespace, and verify a proof on-chain.

## Prerequisites

<CardGroup cols={2}>
  <Card title="Node.js 20+" icon="node-js">
    The app is a Next.js 16 monorepo using npm workspaces.
  </Card>
  <Card title="Sui CLI (optional)" icon="terminal">
    Only needed to build/publish the Move contract yourself.
  </Card>
</CardGroup>

<Note>
Cairn is **mock-first**: real OpenAI / Anthropic / Walrus adapters sit behind interfaces, and the system falls back to deterministic mocks when a key is absent — so it runs offline and a demo never hard-fails on a flaky testnet.
</Note>

## 1. Clone and install

```bash
git clone https://github.com/Enoch208/cairn
cd cairn
npm install
```

## 2. Configure environment

Copy the example env and fill in what you have. With **no keys**, Cairn runs in mock mode; with keys, it runs live.

```bash
cp apps/web/.env.example apps/web/.env.local
```

<CodeGroup>
```bash Live mode
# apps/web/.env.local
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
WALRUS_PUBLISHER=https://publisher.walrus-testnet.walrus.space
WALRUS_AGGREGATOR=https://aggregator.walrus-testnet.walrus.space
CAIRN_MODE=live
```

```bash Mock mode
# apps/web/.env.local
CAIRN_MODE=mock
# no API keys needed — deterministic stubs for LLM + Walrus
```
</CodeGroup>

<Tip>
The on-chain config (package + policy IDs) is read from `deployments/testnet.json`, so you don't need to set it manually to use the public verifier.
</Tip>

## 3. Run the app

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## 4. The five-minute tour

<Steps>
  <Step title="Teach a fact">
    Open **Chat A** (GPT-4o). Use the capture box to store a fact in the `health` namespace — e.g. _"Allergic to penicillin."_ It's written to Walrus as a real blob.
  </Step>
  <Step title="Recall across models">
    Open **Chat B** (Claude). Ask _"Am I allergic to anything?"_ Agent B — a different provider — recalls the fact and renders an **Answer Receipt** showing the memory used, that it was authorized, and that the blob still resolves on Walrus.
  </Step>
  <Step title="Revoke and watch the gate">
    Open **Access** and revoke Agent B's access to `health`. Ask again — the agent truthfully refuses, and the receipt shows `health blocked by policy`. The gate ran **before** the model, so the memory was never fetched.
  </Step>
  <Step title="Anchor and verify">
    On the **Dashboard**, anchor the latest receipt on-chain (one wallet signature). Open `/verify/<objectId>` — with no wallet — and watch Cairn independently confirm the proof against Sui and Walrus.
  </Step>
</Steps>

## 5. Run the tests

```bash
npm test                      # TypeScript: core gate, receipts, digest, verify
cd contracts && sui move test # Move: receipt minting, hash chain, blake2b256
```
